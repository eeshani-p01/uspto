  • Access Profile

Apache Hive is a data warehouse software project built on top of Apache Hadoop for providing data summarization, query, and analysis. Hive gives an SQL-like interface to query data stored in various databases and file systems that integrate with Hadoop. Traditional SQL queries must be implemented in the MapReduce Java API to execute SQL applications and queries over distributed data. Hive provides the necessary SQL abstraction to integrate SQL-like queries (HiveQL) into the underlying Java without the need to implement queries in the low-level Java API. Since most data warehousing applications work with SQL-based querying languages, Hive aids portability of SQL-based applications to Hadoop.

A model that applies Natural Language Processing, Information Extraction, Entity Recognition and Resolution, along with structural information and institutional knowledge, to extract various features from an office action by deep analysis of the document text. Machine learning algorithms are then used to train different regression/predictive models, guided by rich information from Master Review Forms (MRF). These models are adopted in the pipeline to detect different rejections and give clarity scores on 102 and 103 rejections, along with other interesting traits to any incoming office actions.
Leveraging Apache Nifi, BDR Data pipelines bring the Data from PTAB AIA Trials, Appeals and Interference Databases, and store in the HIVE landing tables. As part of the Data pipelines, BDR uses OPSG APIs to bring additional metadata needed the Appeals. Once the data is fetched from multiple sources, pipelines bring the documents from Alfresco. If the document is decision, then it uses TIKA to extract the text from PDFs, and derive the board rulings and issue types.
After the business rules are applied to map the decision types and push the data to SOLR and Elastic Search. APIs and UIs are developed to present the data available in SOLR and Elastic Search.
High Level Capabilities:

Full Text Search capabilities for all the Decision Documents. BDR process extracts the text from the Decision Documents and make the text searchable.

From the PDFs, we extract the Board Rulings, Issue Types (101,102,103.etc) and made them available as Facets in the UI and the same are filterable through APIs.

Search capabilities are powered through SOLR (public) and Elastic Search (Internal).

Data available in BDR can be accessible through APIs and bulk-downs.

Leverage BDR’s Data Science platform, data scientists can build the insights on the data available in the BDR.

Leveraging Apache Nifi, BDR Data pipelines retrieves the Data from TRM and FAST2 databases, and store in the HIVE landing tables. As part of the Data pipelines, BDR uses RBAC and TMNG CMS REST services to retrieve lawcode and Mark images. After the business rules are applied pushes the data to Elastic Search. APIs and UIs are developed to present the data available in Elastic Search.
High Level Capabilities:

BDR-TQR retrieves SOU and PUBs metadata information from TRM Oracle database and First OA and Final OA metadata from FAST Oracle database.

It retrieves lawcode from RBAC and Mark images from TMNG CMS REST service.

Users can place input .csv file by using SFTP, which contains number of application IDs. By using this input file BDR AI API gets the contractor data from the CPC Oracle DB and machine data from the BDR AI for corresponding application IDs and stores the data in two csv files. Users can compare contractor data and machine data, by giving application ID in the CPC WEB.
There is separate batch process for cpccode, cstar and cpccode-cstar to generate .csv files for Machine data and Contractor data.
AI-API directly connects with CPC Oracle DB to get the contractor data on the server jdbc:oracle:thin:@//das-gds.uspto.gov:1610/cpcprod_reporting_gds.cpc_prod.oradbcloud. To get the Machine Data, AI-API connects to Lucene model. For this model, the Lucene indexes and model files are located in the location data/oa/cpc. BDR AI-API also interface with PATI-CDC to get Meta data.
